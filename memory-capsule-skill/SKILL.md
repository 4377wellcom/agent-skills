# 记忆胶囊（Memory Capsule）

将当前对话内容制作成结构化的记忆胶囊文件，保存在 markdown 文件中。
所有胶囊相关文件统一存放在 `胶囊-{agent名称}/` 文件夹中。
胶囊内容采用结构化提取框架，对对话进行深度分析后输出。
当胶囊文件 token 数超过阈值时，追加操作前会自动整合去重，避免内容臃肿。
（默认阈值 4000 token，可通过 `/ai-memory-threshold <数值>` 自定义）

## 胶囊文件夹

所有文件统一组织在 `胶囊-{agent名称}/` 文件夹中，便于管理、迁移和隔离。

### 位置变量与路标文件
技能通过 `_position.json` 路标文件持久化记录胶囊文件夹的实际位置，确保跨新框仍能正确定位：

- **路标位置**：仅存储在 Agent 的**工作目录**（accessible_paths[0]）
- **路标内容**：记录当前 agent 胶囊文件夹的完整路径
- **查找规则**：新框启动时 → 读路标 → 按路标指向找到胶囊文件夹
- **搬家时**：更新路标文件，始终只有最新路标
- 不同 agent 的路径记录相互独立

### 文件夹结构
```
{工作目录}/
├── _position.json         ← 路标文件，记录当前 agent 胶囊的实际路径
│
{胶囊实际位置}/
└── 胶囊-{agent名称}/
    ├── 胶囊-{agent名称}-1.md
    ├── 胶囊-{agent名称}-2.md
    └── _counter.json       # 含版本号和 threshold 字段
```

## 命令

### `/ai-memory [路径]`
基础版 — 在当前对话框创建或追加内容到记忆胶囊。
**重要：所有操作必须先创建 `胶囊-{agent名称}/` 文件夹，胶囊文件和计数器放在文件夹内部，不可松散放置。**

- **首次使用 + 无参数**：
  1. 在工作目录下创建 `胶囊-{agent名称}/` 文件夹
  2. 在文件夹内创建 `_counter.json`（版本 1，threshold 4000）
  3. 在工作目录创建 `_position.json`，写入路径指向刚创建的胶囊文件夹
  4. 分析当前对话 → 写入 `胶囊-{agent名称}-1.md` 到文件夹内

- **首次使用 + 有参数**（如 `/ai-memory D:\我的胶囊`）：
  1. 在指定路径下创建 `胶囊-{agent名称}/` 文件夹
  2. 在文件夹内创建 `_counter.json`（版本 1，threshold 4000）
  3. 在工作目录创建 `_position.json`，写入路径指向指定路径的胶囊文件夹
  4. 分析当前对话 → 写入 `胶囊-{agent名称}-1.md` 到文件夹内

- **已有胶囊 + 无参数**：
  1. 按路标定位到当前胶囊文件夹
  2. 检查 token 数 → 超过阈值先整合再追加
  3. 旧内容 + 新对话 → 合并 → 版本+1 → 写入文件夹内

- **已有胶囊 + 有参数**（搬家）：
  1. 按路标定位到当前位置
  2. 将整个 `胶囊-{agent名称}/` 文件夹搬移到指定路径下
  3. 更新工作目录下的 `_position.json` 指向新路径
  4. 在新位置执行追加操作（token 检查 → 合并 → 版本+1）

### `/ai-memory-new`
整合版 — 将当前记忆胶囊中所有框的内容重新整合成一整篇。
- 在当前位置变量的胶囊文件夹内操作
- 读取胶囊全部内容 + 当前对话 → AI 重新整合、优化为一整篇更精炼的版本
- 文件名更新为 `胶囊-{agent名称}-{n}.md`（n=当前版本号）
- 删除旧的胶囊文件

### `/ai-memory-threshold [数值]`
阈值管理 — 查看或设置自动整合的 token 阈值。
- **无参数**：显示当前阈值设置
- **有参数**（如 `/ai-memory-threshold 6000`）：将阈值更新为指定数值
- 阈值保存在 `_counter.json` 中
- 默认值：4000 token

### `/ai-memory-read`
读取版 — 将最新记忆胶囊内容读入上下文（AI 可基于胶囊内容回答问题），**不在对话框中展示胶囊全文**。
- 在当前位置变量的胶囊文件夹内读取最新文件到上下文
- 如果没有任何胶囊，告知用户尚未创建记忆胶囊
- 回复简短确认消息，不输出胶囊正文
- 支持 `--full` 参数（`/ai-memory-read --full`）→ 在对话框中展示完整胶囊内容
- 支持 `list` 参数（`/ai-memory-read list`）→ 列出当前 agent 的胶囊状态（版本号、存储路径、最后更新时间）

## 结构化提取框架

胶囊内容采用 conversation-extractor 的结构化提取方法，对当前对话进行深度分析后按以下类别提取信息：

| 类别 | 提取内容 |
|------|---------|
| **核心主题** | 对话主要讨论什么？整体目标是什么？ |
| **关键决策** | 做出了哪些决定？ |
| **行动项** | 需要完成什么事？ |
| **待解问题** | 哪些问题悬而未决？ |
| **重要事实/数据** | 提到的具体数字、日期、名称、链接等客观信息 |
| **参与方观点** | 各方表达了什么观点或诉求？ |
| **总结** | 对话的最终结论或下一步方向 |

### 提取原则
1. **忠实于原文** — 不添加对话中没有的信息，不臆测
2. **全面覆盖** — 尽可能覆盖所有重要信息，不遗漏关键点
3. **结构化呈现** — 使用清晰的 Markdown 层级组织内容
4. **引用来源** — 对于重要信息，标注大致出自哪部分对话
5. **如果某类别没有信息则省略**，不要输出空表格

## 文件结构

### 路标文件 `_position.json`
位于 Agent 的工作目录中，记录当前 agent 的胶囊文件夹实际路径。
```json
{
  "agent-name": "E:\\work\\记忆胶囊\\胶囊-agent-name"
}
```
| 字段 | 说明 |
|------|------|
| `agent-name` | agent 名称 → 胶囊文件夹的完整绝对路径 |

**查找逻辑：**
1. 在工作目录找 `_position.json`
2. 找到且非空 → 按路径定位胶囊文件夹
3. 找到但内容缺失 → 在工作目录下找 `胶囊-{agent}/` 文件夹兜底
4. 找不到 `_position.json` → 在工作目录下找 `胶囊-{agent}/` 文件夹
5. 都没有 → 首次使用，在工作目录创建

### 计数器
`_counter.json` — 位于胶囊文件夹内，追踪当前 agent 的版本号和阈值设置。
```json
{
  "agent-name": 2,
  "threshold": 4000
}
```
| 字段 | 说明 | 默认值 |
|------|------|--------|
| `agent-name` | 当前版本号 | 1 |
| `threshold` | 自动整合的 token 阈值 | 4000 |
- 没有计数器文件 → 新建，版本 1，阈值 4000
- 有计数器文件 → 读取当前值，+1 后写入新文件

### 文件名
`胶囊-{agent名称}-{n}.md`
- `n` = 当前版本号（每次调用递增）
- 始终只有一个胶囊文件

### 胶囊内容格式
每次调用按结构化提取框架生成内容，写入胶囊文件：
```markdown
# 记忆胶囊 - {agent名称} - 第{n}版

## 📅 元信息
- **创建时间**: YYYY-MM-DD
- **最后更新**: YYYY-MM-DD

## 📦 第1次调用 - YYYY-MM-DD

### 📋 对话摘要

#### 一、核心主题
> [用 1-3 句话概括本次对话的核心内容]

#### 二、关键决策
| # | 决策内容 | 提出方 | 状态 |
|---|---------|--------|------|
| 1 | ... | ... | ✅ 已确定 / ⏳ 待确认 |

#### 三、行动项
| # | 行动内容 | 负责人 | 截止时间 | 优先级 |
|---|---------|--------|---------|--------|
| 1 | ... | ... | ... | 🔴高/🟡中/🟢低 |

#### 四、待解决问题
| # | 问题描述 | 需要谁来确认 | 备注 |
|---|---------|------------|------|

#### 五、重要事实与数据
- [关键数据点]

#### 六、参与方观点
| 参与方 | 核心观点/立场 | 备注 |
|--------|--------------|------|

#### 七、总结
> [最终结论或下一步方向]

## 📦 第2次调用 - YYYY-MM-DD
[同上结构化格式]
```

## 工作流程

### 路标查找规则
在所有操作开始前，按以下顺序确定胶囊文件夹位置：

1. **读路标优先**：在工作目录下用 Read 工具读取 `_position.json`，解析当前 agent 的路径
   → 如果路径存在（文件夹实际存在），直接定位

2. **兜底查找**：路标文件找不到或内容缺失时，在工作目录下查找 `胶囊-{agent名称}/` 文件夹
   → 找到则直接使用该路径

3. **首次使用**：以上都没有 → 按首次使用处理
   → 如果用户指定了路径，在指定路径创建胶囊文件夹
   → 如果用户没指定路径，在工作目录创建胶囊文件夹

**文件夹结构强制规则（所有场景通用）：**
- ⚠️ **必须先创建 `胶囊-{agent名称}/` 文件夹，再在文件夹内创建胶囊文件和计数器**
- ⚠️ **绝不允许将胶囊文件或计数器直接放在目录下，必须放在文件夹内部**
- ⚠️ **`_position.json` 路标文件始终存放在工作目录下，不在胶囊文件夹内**

### 场景 1：第一次使用 `/ai-memory`（无参数）
1. 确认：工作目录无路标文件、无胶囊文件夹
2. **在工作目录下创建 `胶囊-{agent名称}/` 文件夹**（必须先建文件夹）
3. 在胶囊文件夹内创建 `_counter.json`，版本 1，threshold 4000
4. 在工作目录创建 `_position.json`，写入 `{"agent-name": "工作目录/胶囊-agent-name"}`
5. 读取当前对话，按结构化提取框架分析
6. **在胶囊文件夹内**创建 `胶囊-{agent名称}-1.md`，写入结构化提取结果

### 场景 1b：第一次使用 `/ai-memory`（带路径参数）
1. 确认：工作目录无路标文件、无胶囊文件夹
2. **在指定路径下创建 `胶囊-{agent名称}/` 文件夹**（必须先建文件夹）
3. 在胶囊文件夹内创建 `_counter.json`，版本 1，threshold 4000
4. 在工作目录创建 `_position.json`，写入 `{"agent-name": "指定路径/胶囊-agent-name"}`
5. 读取当前对话，按结构化提取框架分析
6. **在胶囊文件夹内**创建 `胶囊-{agent名称}-1.md`，写入结构化提取结果

### 场景 2：后续使用 `/ai-memory`（带路径搬家）
1. 按路标（或兜底查找）定位到当前位置，确认胶囊文件夹存在
2. **搬家**：将整个 `胶囊-{agent名称}/` 文件夹移动到指定路径下
3. **更新路标**：修改工作目录下 `_position.json` 中当前 agent 的路径指向为新位置
4. 读取新位置下的 `_counter.json`，版本 N，threshold T
5. **检查 token**：
   - 超过 T → 先 `/ai-memory-new` 整合（版本+1）
   - 未超过 → 直接继续
6. 读取旧胶囊 + 当前对话框 → 结构化分析 → 合并
7. **在胶囊文件夹内**写入 `胶囊-{agent名称}-{N+1}.md`，删除旧文件
8. 更新 `_counter.json` 为 N+1

### 场景 3：后续使用 `/ai-memory`（原地追加）
1. 按路标定位到当前胶囊文件夹，读取 `_counter.json`（版本 N，threshold T）
2. **检查 token**：
   - 超过 T → 先 `/ai-memory-new` 整合（版本+1）
   - 未超过 → 直接继续
3. 读取旧胶囊 + 当前对话框 → 结构化分析 → 合并
4. 写入 `胶囊-{agent名称}-{N+1}.md`，删除旧文件
5. 更新 `_counter.json` 为 N+1

### 场景 4：使用 `/ai-memory-new`
1. 按路标定位到当前胶囊文件夹，读取全部内容
2. 读取当前对话框内容，按结构化提取框架分析
3. AI 将所有信息重新整合、去重、优化为一整篇更精炼的版本
4. 写入 `胶囊-{agent名称}-{N+1}.md`，删除旧文件
5. 更新 `_counter.json`

### 场景 5：使用 `/ai-memory-read`
1. 按路标定位到当前胶囊文件夹，读取 `_counter.json`，获取最新版本号 N
2. 查找 `胶囊-{agent名称}-{N}.md`
3. 文件不存在 → 提示用户尚无记忆胶囊
4. 文件存在 → 读取内容到上下文，回复简短确认
5. 如果使用 `--full` 参数 → 在对话框中完整展示
6. 如果使用 `list` 参数 → 显示胶囊概要信息

## 核心原则
- 🛡️ **不覆盖用户已有文件** — 操作前检查文件是否存在
- 🛡️ **计数器准确** — 每次操作后必须更新 `_counter.json`
- 🛡️ **保留历史** — 旧文件删除前确保新文件已写入成功
- 🛡️ **搬家不改内容** — 指定路径时仅移动胶囊文件夹到新位置，内容、版本号不变
- 🛡️ **位置变量隔离** — 不同 agent 的位置变量相互独立
- 🛡️ **自动阈值整合** — 追加前检查 token 数，超过阈值自动整合去重后再追加（默认 4000）
- 🛡️ **路标持久化** — `_position.json` 仅存于工作目录，跨新框仍可定位胶囊实际位置

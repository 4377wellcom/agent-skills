# 记忆胶囊（Memory Capsule）功能介绍

> 版本：v1.6.0 | 最后更新：2026-05-05

---

## 一、概述

记忆胶囊是一款专为 Cherry Studio Agent 设计的结构化对话记忆工具。它能将 Agent 与用户的对话内容按 7 个结构化类别进行深度分析，持久化保存为 Markdown 文件，并支持跨会话读取、整合优化和位置迁移。

### 核心能力

| 能力 | 说明 |
|------|------|
| 制作胶囊 | 将当前对话按结构化框架提取，保存为胶囊文件 |
| 追加记忆 | 在已有胶囊基础上追加新对话内容，版本号自动递增 |
| 整合优化 | 将所有胶囊内容重新整合、去重、提炼为更精炼的版本 |
| 读取记忆 | 将胶囊内容读入上下文，AI 可基于记忆回答问题 |
| 自动整合 | 当胶囊内容超过 token 阈值时，先自动整合再追加，避免臃肿 |
| 搬家迁移 | 可指定新路径，整个胶囊文件夹自动搬移到目标位置 |
| 跨框定位 | 通过路标文件持久化记录位置，新会话也能找到胶囊 |

---

## 二、胶囊文件夹

所有文件统一归入 `胶囊-{agent名称}/` 文件夹中。

### 文件夹结构

```
{工作目录}/
├── _position.json              ← 路标文件

{胶囊实际位置}/
└── 胶囊-{agent名称}/
    ├── 胶囊-{agent名称}-1.md   ← 胶囊文件（版本号递增）
    ├── 胶囊-{agent名称}-2.md
    └── _counter.json           ← 计数器和阈值设置
```

### 路标文件（_position.json）

位于 Agent 的**工作目录**，持久化记录胶囊文件夹的实际路径。

```json
{
  "l7v33ohxo": "E:\\work\\记忆胶囊\\胶囊-l7v33ohxo"
}
```

**跨新框定位逻辑：**
1. 在工作目录读取 `_position.json`
2. 有记录 → 按路径定位胶囊文件夹
3. 无记录 → 在工作目录找 `胶囊-{agent}/` 文件夹兜底
4. 都没有 → 按首次使用在工作目录创建

## 三、命令详解

### 1. `/ai-memory [路径]`

**基础版 — 创建或追加记忆胶囊**

> ⚠️ **重要：所有操作必须先创建 `胶囊-{agent名称}/` 文件夹，胶囊文件和计数器放在文件夹内部，不可松散放置。**

| 场景 | 行为 |
|------|------|
| 首次使用 + 无参数 | 在工作目录创建 `胶囊-agent/` → 写胶囊 + 计数器和路标 |
| 首次使用 + 指定路径 | 在指定路径创建 `胶囊-agent/` → 写胶囊 + 计数器 → 路标指向该路径 |
| 已有胶囊 + 无参数 | 按路标定位 → 追加（token 超阈值先自动整合） |
| 已有胶囊 + 指定路径 | 搬整个 `胶囊-agent/` 到指定路径 → 更新路标 → 追加 |

**自动整合：** 追加前检查胶囊文件 token 数，超过阈值（默认 4000）时，先自动执行整合去重，再追加新内容。

**输出：** 回复简短确认消息，不输出胶囊全文。

---

### 2. `/ai-memory-new`

**整合版 — 将全部胶囊内容重新整合为一篇**

- 读取当前胶囊文件夹的所有历史内容 + 当前对话
- AI 自动合并同类项、去重冗余、提炼共性
- 写入新文件（版本号+1），删除旧文件
- 适合胶囊内容变得冗长时手动调用

---

### 3. `/ai-memory-threshold [数值]`

**阈值管理 — 查看或修改自动整合的触发阈值**

| 用法 | 行为 |
|------|------|
| `/ai-memory-threshold` | 显示当前阈值 |
| `/ai-memory-threshold 6000` | 修改为 6000 token |

阈值保存在 `_counter.json` 中，默认 4000。

---

### 4. `/ai-memory-read [参数]`

**读取版 — 将胶囊内容读入上下文**

| 参数 | 行为 |
|------|------|
| 无参数 | 读取到上下文，回复简短确认，AI 可基于记忆回答 |
| `--full` | 在对话框中展示完整胶囊内容 |
| `list` | 显示胶囊状态概要（版本号、路径、更新时间） |

## 四、结构化提取框架

每次制作胶囊时，AI 按以下 7 个类别对对话进行深度分析：

| 类别 | 说明 |
|------|------|
| 核心主题 | 对话主要讨论什么？整体目标是什么？ |
| 关键决策 | 做出了哪些决定，由谁提出 |
| 行动项 | 需要完成什么事，负责人和优先级 |
| 待解问题 | 哪些问题悬而未决 |
| 重要事实/数据 | 具体数字、日期、名称、链接等客观信息 |
| 参与方观点 | 各方表达了什么观点或诉求 |
| 总结 | 对话的最终结论或下一步方向 |

**提取原则：** 忠实于原文、全面覆盖、结构化呈现、引用来源、无信息时不输出空表。

## 五、工作流程

### 5.1 第一次使用

```
用户: /ai-memory
  → 在工作目录创建 胶囊-agent/ 文件夹
  → 创建 _position.json 路标
  → 创建 _counter.json（版本 1，threshold 4000）
  → 分析当前对话 → 写入 胶囊-agent-1.md
  → 回复: 已制作记忆胶囊
```

### 5.2 追加记忆

```
用户: /ai-memory
  → 读 _position.json 定位胶囊文件夹
  → 检查 token 数:
      > 4000 → 先整合去重（版本+1）
      ≤ 4000 → 直接追加
  → 旧内容 + 新对话 → 合并写入 → 版本+1
  → 回复: 已追加记忆胶囊
```

### 5.3 搬家迁移

```
用户: /ai-memory D:\新位置
  → 将 胶囊-agent/ 文件夹搬到 D:\新位置
  → 更新 _position.json 指向新路径
  → 执行追加操作（token 检查 → 合并 → 版本+1）
  → 回复: 已搬移并追加
```

### 5.4 整合优化

```
用户: /ai-memory-new
  → 读取胶囊全部内容 + 当前对话
  → AI 重新整合、去重、优化
  → 写入新版本，删除旧文件
  → 回复: 已重新整合
```

### 5.5 读取记忆

```
用户: /ai-memory-read
  → 按路标定位 → 读取最新胶囊文件
  → 读入上下文
  → 回复: 已读取记忆胶囊，可以基于记忆回答
  → 用户追问: 之前讨论过什么？
  → AI 根据记忆回答
```

## 六、核心原则

| 原则 | 说明 |
|------|------|
| 不覆盖用户文件 | 操作前检查文件是否存在 |
| 计数器准确 | 每次操作后更新 `_counter.json` |
| 保留历史 | 旧文件删除前确保新文件已写入成功 |
| 搬家不改内容 | 移动文件夹时，内容、版本号不变 |
| 位置变量隔离 | 不同 Agent 的路标和胶囊相互独立 |
| 自动阈值整合 | token 超过阈值自动整合（默认 4000） |
| 路标持久化 | `_position.json` 存于工作目录，跨新框可定位 |

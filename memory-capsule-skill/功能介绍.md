# 记忆胶囊 Skill 功能介绍

Memory Capsule — 结构化对话记忆工具

版本：1.5.0 | 最后更新：2026-05-05

---

## 1. 概述

记忆胶囊（Memory Capsule）是一个结构化对话记忆工具，专为 Cherry Studio Agent 设计。它的核心功能是将 Agent 与用户的对话内容，按照 7 个结构化类别进行深度分析，然后持久化保存到 Markdown 文件中。

### 主要能力

| 能力 | 说明 |
|------|------|
| **制作记忆胶囊** | 将当前对话内容按结构化框架提取并保存为胶囊文件 |
| **追加记忆** | 在已有胶囊基础上追加新对话的结构化内容，版本号自动递增 |
| **整合优化** | 将所有已保存的胶囊内容重新整合、去重、提炼为更精炼的版本 |
| **读取记忆** | 将胶囊内容读入上下文，AI 可基于记忆回答问题 |
| **自动阈值整合** | 当胶囊内容超过 token 阈值时，自动先整合再追加，避免内容臃肿 |
| **搬家迁移** | 可指定新路径，整个胶囊文件夹自动搬移到目标位置 |
| **位置变量** | 不同 Agent 的位置变量相互独立，互不干扰 |

当前版本：v1.5.0（2026-05-05）

## 2. 胶囊文件夹与位置变量

### 2.1 胶囊文件夹

所有胶囊相关文件统一归入 `胶囊-{agent名称}/` 文件夹中，便于管理、迁移和隔离。

### 2.2 位置变量

技能内部维护一个**位置变量**，记录胶囊文件夹所在的外层路径：

- **初始状态** → 位置变量指向 Agent 的工作目录
- 用户通过 `/ai-memory D:\路径` 指定路径后 → 位置变量更新为该路径
- **不同 Agent 的位置变量相互独立**，互不干扰
- 搬家仅移动文件夹位置，**文件内容和版本号不变**

### 2.3 文件夹结构示意

```
{位置变量}/
└── 胶囊-{agent名称}/
    ├── 胶囊-{agent名称}-1.md
    ├── 胶囊-{agent名称}-2.md
    └── _counter.json   # 含版本号和 threshold 字段
```

## 3. 命令详解

### 3.1 `/ai-memory [路径]` — 基础创建/追加

**触发方式：** `/ai-memory`、记忆胶囊、胶囊、保存记忆、整理对话

**功能：** 在当前对话框创建或追加内容到记忆胶囊。

**行为：**

- **无参数：** 在当前位置变量的目录下操作
- **有参数**（如 `D:\我的胶囊`）：先搬移整个胶囊文件夹到指定路径下，更新位置变量再操作
- **首次使用：** 创建胶囊文件夹和计数器文件，写入结构化提取结果
- **非首次使用：** 读取旧胶囊 → 追加新对话框内容 → 版本号+1
- **自动整合：** 追加前检查 token 数，超过 threshold 则先执行 `/ai-memory-new` 整合去重

### 3.2 `/ai-memory-new` — 整合优化

**触发方式：** `/ai-memory-new`

**功能：** 将记忆胶囊所有框的内容 + 当前对话框 → AI 重新整合、去重、优化为一整篇更精炼的版本。

**行为：**

1. 读取当前位置胶囊文件夹内的全部历史内容
2. 读取当前对话框的结构化提取结果
3. AI 自动合并同类项、提炼共性、去重冗余
4. 写入新文件（版本号+1），删除旧文件

### 3.3 `/ai-memory-threshold [数值]` — 阈值管理

**触发方式：** `/ai-memory-threshold`

**功能：** 查看或设置自动整合的 token 阈值。

**行为：**

- **无参数：** 显示当前阈值设置（如"当前阈值：4000 token"）
- **有参数：** 将阈值更新为指定数值（如 `/ai-memory-threshold 6000`）
- 阈值保存在 `_counter.json` 的 `threshold` 字段中
- **默认值：** 4000 token

### 3.4 `/ai-memory-read` — 读取胶囊

**触发方式：** `/ai-memory-read`、读取记忆

**功能：** 将最新记忆胶囊内容读入上下文（AI 可基于胶囊内容回答问题），不在对话框中展示胶囊全文。

**参数：**

- 无参数 → 读入上下文，回复简短确认
- `--full` → 读取后在对话框中展示完整胶囊内容
- `list` → 列出当前 agent 的胶囊状态（版本号、存储路径、最后更新时间）

## 4. 结构化提取框架

胶囊内容采用 **conversation-extractor** 的结构化提取方法，对当前对话进行深度分析后按 7 个类别提取信息：

| 类别 | 提取内容 |
|------|---------|
| **核心主题** | 对话主要讨论什么？整体目标是什么？ |
| **关键决策** | 做出了哪些决定？ |
| **行动项** | 需要完成什么事？ |
| **待解问题** | 哪些问题悬而未决？ |
| **重要事实/数据** | 提到的具体数字、日期、名称、链接等客观信息 |
| **参与方观点** | 各方表达了什么观点或诉求？ |
| **总结** | 对话的最终结论或下一步方向 |

### 提取原则

1. **忠实于原文** — 不添加对话中没有的信息，不臆测
2. **全面覆盖** — 尽可能覆盖所有重要信息，不遗漏关键点
3. **结构化呈现** — 使用清晰的 Markdown 层级组织内容
4. **引用来源** — 对于重要信息，标注大致出自哪部分对话
5. 如果某类别没有信息则省略，不要输出空表格

### 胶囊输出格式示例

```markdown
# 记忆胶囊 - {agent名称} - 第{n}版

## 📅 元信息
- **创建时间**: YYYY-MM-DD
- **最后更新**: YYYY-MM-DD

## 📦 第1次调用 - YYYY-MM-DD

### 📋 对话摘要

#### 一、核心主题
> [用 1-3 句话概括本次对话的核心内容]

#### 二、关键决策
| # | 决策内容 | 提出方 | 状态 |
|---|---------|--------|------|

#### 三、行动项
| # | 行动内容 | 负责人 | 截止时间 | 优先级 |
|---|---------|--------|---------|--------|

#### 四、待解决问题
| # | 问题描述 | 需要谁来确认 | 备注 |
|---|---------|------------|------|

#### 五、重要事实与数据
- [关键数据点]

#### 六、参与方观点
| 参与方 | 核心观点/立场 | 备注 |
|--------|--------------|------|

#### 七、总结
> [最终结论或下一步方向]
```

## 5. 文件结构

### 5.1 `_counter.json`（计数器）

位于胶囊文件夹内，追踪版本号和阈值设置。

```json
{
  "agent-name": 3,
  "threshold": 4000
}
```

- 没有计数器文件 → 新建，版本 1，阈值 4000
- 有计数器文件 → 读取当前值，+1 后写入新文件

### 5.2 胶囊文件命名规则

`胶囊-{agent名称}-{n}.md`

- `n` = 当前版本号（每次调用递增）
- 始终只有一个胶囊文件（旧版本会被新版本覆盖）
- 通过 `/ai-memory-new` 整合后，版本号+1，内容重写

## 6. 工作流程详解

### 6.1 位置变量查找规则

所有操作开始前，先按以下规则确定当前位置：

1. 查找 `胶囊-{agent名称}/` 文件夹是否存在于上次记录的位置
2. 如果找到 → 该路径即为当前位置
3. 如果没找到 → 位置变量重置为工作目录
4. 如果指定了路径参数 → 执行搬家流程，再操作

### 6.2 第一次使用 `/ai-memory`

1. 位置变量为工作目录（或用户指定路径）
2. 在位置变量下创建 `胶囊-{agent名称}/` 文件夹
3. 创建 `_counter.json`，版本设为 1，threshold 设为 4000
4. 读取当前对话内容，按结构化提取框架进行深度分析
5. 创建 `胶囊-{agent名称}-1.md`，写入结构化提取结果
6. 回复简短确认消息（不输出内容摘要）

### 6.3 后续使用 `/ai-memory`（带路径搬家）

1. 当前位置已有胶囊文件夹
2. **搬家：** 将整个 `胶囊-{agent名称}/` 文件夹移动到指定路径下
3. 位置变量更新为指定路径
4. 读取 `_counter.json`，当前版本为 N，threshold 为 T
5. 检查当前胶囊文件的 token 数：
   - 如果 token 数 > T → **先执行 `/ai-memory-new` 整合**（版本号+1）再继续
   - 如果 token 数 ≤ T → 直接继续
6. 读取旧胶囊内容 + 当前对话框 → 结构化分析 → 合并
7. 写入 `胶囊-{agent名称}-{N+1}.md`，删除旧文件
8. 更新 `_counter.json` 版本为 N+1

### 6.4 后续使用 `/ai-memory`（原地追加）

1. 位置变量不变，读取 `_counter.json`（版本 N，threshold T）
2. 检查当前胶囊文件的 token 数：
   - 如果 token 数 > T → 先执行 `/ai-memory-new` 整合再继续
   - 如果 token 数 ≤ T → 直接继续
3. 读取旧胶囊内容 + 当前对话框 → 结构化分析 → 合并
4. 写入 `胶囊-{agent名称}-{N+1}.md`，删除旧文件
5. 更新 `_counter.json` 版本为 N+1

### 6.5 使用 `/ai-memory-new`（整合）

1. 读取当前位置胶囊文件夹内的全部内容
2. 读取当前对话框内容，按结构化提取框架分析
3. AI 将所有信息重新整合、去重、优化为一整篇更精炼的版本
4. 写入 `胶囊-{agent名称}-{N+1}.md`，删除旧文件
5. 更新 `_counter.json`

### 6.6 使用 `/ai-memory-read`（读取）

1. 读取当前位置胶囊文件夹内的 `_counter.json`，获取最新版本号 N
2. 查找 `胶囊-{agent名称}-{N}.md`
3. 文件不存在 → 提示用户尚无记忆胶囊
4. 文件存在 → 读取内容到上下文，回复简短确认
5. 使用 `--full` 参数 → 在对话框中完整展示
6. 使用 `list` 参数 → 显示胶囊概要信息

## 7. 核心原则

- **不覆盖用户已有文件** — 操作前检查文件是否存在
- **计数器准确** — 每次操作后必须更新 `_counter.json`
- **保留历史** — 旧文件删除前确保新文件已写入成功
- **搬家不改内容** — 指定路径时仅移动胶囊文件夹到新位置，内容、版本号不变
- **位置变量隔离** — 不同 Agent 的位置变量相互独立
- **自动阈值整合** — 追加前检查 token 数，超过阈值自动整合去重后再追加（默认 4000）

## 9. 常见问题

**Q: 胶囊文件存在哪里？**
默认存放在 Agent 的工作目录下的 `胶囊-{agent名称}/` 文件夹中。可以通过 `/ai-memory 路径` 命令搬移到任意位置，位置变量会自动更新。

**Q: 为什么有多个版本（v1、v2、v3）？**
每次调用 `/ai-memory` 都会产生一个新版本。版本号代表胶囊的迭代次数。可以通过 `/ai-memory-new` 整合去重，合并同类项。

**Q: 什么是 token 阈值？有什么用？**
阈值控制自动整合的触发条件。当胶囊文件 token 数超过阈值时，追加操作前会自动整合去重。默认 4000 token，可通过 `/ai-memory-threshold` 自定义。

**Q: 不同 Agent 的胶囊会相互影响吗？**
不会。每个 Agent 有自己的位置变量、独立的胶囊文件夹和计数器文件，完全隔离。

**Q: 如何查看胶囊里记了什么？**
使用 `/ai-memory-read` 将胶囊读入上下文，AI 可基于记忆回答问题。使用 `/ai-memory-read --full` 可直接查看胶囊全文。

**Q: 搬家时胶囊内容会变吗？**
不会。搬家仅将整个胶囊文件夹移动到新位置，文件内容、版本号、计数器均保持不变。

**Q: 输出为什么这么简短？**
这是设计选择。所有命令只回复简短确认消息，不输出内容摘要，保持对话框整洁。想看完整内容用 `--full` 参数。
